# Словарь терминов

Данная страница поможет лучше понять термины, используемые в документации библиотеки DITranquillity.

## Основные понятия

### DI контейнер

Главный объект библиотеки (`DIContainer`) для регистрации компонентов и создания экземпляров классов

### Компонент

Набор метаданных, описывающих способ создания объекта: тип, инициализатор, время жизни, альтернативные типы и т.д.


### [Регистрация](core/registration_and_service.md)

Код, описывающий способ создания объекта. При регистрации создаются компоненты

### Зависимость

Контракт между объектами: для создания одного объекта требуется создать другие


### [Внедрение](core/injection.md)

Процесс передачи зависимостей в объект при его создании или после

### Resolve

Процесс получения объекта из контейнера (синхронный или асинхронный)

## Компоненты и регистрация

### [Рутовый компонент](core/registration_and_service.md#рутовый-компонент)

Компонент, помеченный `.root()`, указывающий на точку входа в граф зависимостей. Может быть не один

### Альтернативный тип
Тип, добавленный через `.as()`, позволяющий получать компонент по протоколу или базовому классу

### Приоритет
Уровень компонента (`default` или `test`), определяющий какой компонент будет выбран при наличии нескольких кандидатов

## [Время жизни](core/scope_and_lifetime.md)

Период от создания экземпляра до его уничтожения

Варианты:
* prototype - Новый экземпляр при каждом запросе (по умолчанию)
* objectGraph - Один экземпляр в рамках одного графа объектов
* perContainer - Один экземпляр на контейнер (синглтон в рамках контейнера)
* perRun - Один экземпляр на запуск приложения
* single - Глобальный синглтон, общий для всех контейнеров
* custom - Пользовательское время жизни через `DIScope`

### [DIScope](core/scope_and_lifetime.md#пользовательское-время-жизни)
 
Класс для создания пользовательских областей видимости

## Модульность

Протокол для группировки связанных регистраций

### [Фреймворк (DIFramework)](core/modular.md#Фреймворк)
Протокол для объединения нескольких частей в модуль

## [Модификаторы](core/modificated_injection.md) и специальные типы
Функции (`by(tag:on:)`, `many()`, `arg()`) для изменения поведения внедрения

Варианты:
* by(tag:) - Маркер для фильтрации компонентов при resolve
* many() - Модификатор для получения всех реализаций типа как массива
* arg() - Модификатор для передачи аргумента при resolve

## Отложенное внедрение

Фабрика для создания новых экземпляров по требованию

### AsyncLazy

Асинхронная версия Lazy для Swift Concurrency

### AsyncProvider

Асинхронная версия Provider для Swift Concurrency

## Graph API

### [DIGraph](graph/get_graph.md)
Граф зависимостей, представленный списком смежности

### DIVertex

Вершина графа: компонент, аргумент или неизвестный тип

### DIEdge

Ребро графа, описывающее зависимость между вершинами

### DICycle 

Структура, описывающая найденный цикл в графе

### [Валидация](graph/graph_validation.md)

Проверка графа на корректность конфигурации
